# 交易回測系統 - 使用說明

## 啟動方式

```bash
cd E:/ClaudeCode/Trade
pip install -r requirements.txt
python app.py
```

瀏覽器開啟 http://127.0.0.1:8050

## 系統架構

系統分為三個主要功能頁籤：

| 頁籤 | 功能 |
|------|------|
| 資料下載 | 下載個股/期貨歷史資料並匯出 CSV |
| 選股 | 透過基本面指標篩選標的（支援美股、台股上市、台股上櫃） |
| 回測 | 設定技術分析策略並執行回測 |

## 一、資料下載

### 資料來源

| 來源 | 說明 | 標的範例 |
|------|------|---------|
| yfinance | 美股、期貨、加密貨幣、指數 | `AAPL`, `ES=F`, `BTC-USD`, `^GSPC` |
| TWSE 台灣證交所 | 台灣上市股票 | `2330` (台積電) |
| TPEx 櫃買中心 | 台灣上櫃股票 | `6547` |

### 操作步驟

1. **選擇資料來源** - 點選 yfinance / TWSE / TPEx
2. **輸入標的** - 從快速選擇清單中選擇，或手動輸入代碼
3. **設定日期範圍** - 選擇起訖日期
4. **選擇 K 線週期** (僅 yfinance)：1分/5分/15分/30分/1小時/日/週/月
5. **預覽資料** - 點擊「預覽資料」查看 K 線圖和資料表
6. **下載 CSV** - 點擊「下載 CSV」，檔案自動儲存至專案 `data/csv/` 資料夾
7. **送入回測** - 點擊「送入回測 →」直接將已下載資料帶入回測頁

### 快速選擇清單

- ES=F (S&P 500 E-mini)、NQ=F (Nasdaq 100)、YM=F (Dow Jones)
- GC=F (黃金)、CL=F (原油 WTI)、SI=F (白銀)
- 6E=F (歐元)、6J=F (日圓)、BTC-USD (比特幣)

## 二、選股篩選

透過基本面財務指標篩選股票，找出符合條件的標的。支援美股和台股線上即時篩選。

### 股票池來源

| 選項 | 說明 |
|------|------|
| 美股範例 (30檔) | 預設 S&P 500 中 30 檔常見股票 |
| **台股上市 (TWSE)** | 從台灣證交所 open API 取得全部上市公司清單，透過 yfinance 抓基本面 |
| **台股上櫃 (TPEx)** | 從櫃買中心 open API 取得全部上櫃公司清單，透過 yfinance 抓基本面 |
| **台股全部 (上市+上櫃)** | 合併上市+上櫃，**股票數最多，速度最慢** |
| 自訂清單 | 手動輸入代碼，以逗號分隔；純數字代碼自動補 `.TW`（如 `2330` → `2330.TW`） |
| CSV 匯入 | 上傳包含基本面資料的 CSV 檔案 |

> ⚠️ 台股上市約 900+ 支、上櫃約 800+ 支，建議先加篩選條件再執行，或使用自訂清單縮小範圍。

### 基本面指標分組

#### 估值

| 指標 | 欄位名 | 說明 |
|------|--------|------|
| 本益比 | PE | 股價 / 每股盈餘 |
| 股價淨值比 | PB | 股價 / 每股淨值 |
| 殖利率 | DividendYield | 年度股利 / 股價 (%) |
| 市值 | MarketCap | 市值（億，台股為億台幣，美股為億美元） |

#### 獲利能力

| 指標 | 欄位名 | 說明 |
|------|--------|------|
| EPS | EPS | 每股盈餘 |
| ROE | ROE | 股東權益報酬率 (%) |
| 毛利率 | GrossMargin | 毛利 / 營收 (%) |
| 營業利益率 | OperatingMargin | 營業利益 / 營收 (%) |
| 淨利率 | NetMargin | 淨利 / 營收 (%) |

#### YOY 年增率（全年）

| 指標 | 欄位名 | 說明 | 來源 |
|------|--------|------|------|
| 營收年增率 | RevenueGrowth | TTM 營收 YOY (%) | yfinance info |
| 獲利年增率 | EarningsGrowth | TTM EPS YOY (%) | yfinance info |
| 毛利率年增 | GrossMarginYOY | 毛利率 YOY 變化 (pp) | 從年報計算 |
| 營益率年增 | OperatingMarginYOY | 營業利益率 YOY 變化 (pp) | 從年報計算 |
| 淨利率年增 | NetMarginYOY | 淨利率 YOY 變化 (pp) | 從年報計算 |
| 淨利年增率 | NetIncomeYOY | 淨利絕對金額 YOY (%) | 從年報計算 |
| **三率三升（年）** | **ThreeRatesUp** | **毛利率、營益率、淨利率三者 YOY 同升則為 1，否則為 0** | 計算欄位 |

#### YOY 年增率（單季）

| 指標 | 欄位名 | 說明 | 來源 |
|------|--------|------|------|
| 單季營收年增 | RevenueYOY_Q | 最近單季 vs 同期 YOY (%) | yfinance info |
| 單季獲利年增 | EarningsYOY_Q | 最近單季獲利 YOY (%) | yfinance info |
| 單季毛利率年增 | GrossMarginYOY_Q | 單季毛利率 YOY 變化 (pp) | 從季報計算 |
| 單季營益率年增 | OperatingMarginYOY_Q | 單季營業利益率 YOY 變化 (pp) | 從季報計算 |
| 單季淨利率年增 | NetMarginYOY_Q | 單季淨利率 YOY 變化 (pp) | 從季報計算 |
| **三率三升（季）** | **ThreeRatesUp_Q** | **毛利率、營益率、淨利率單季三者 YOY 同升則為 1，否則為 0** | 計算欄位 |

### 三率三升說明

「三率三升」為台股常用選股策略：毛利率、營業利益率、淨利率三者相較去年同期全部提升，代表企業獲利品質持續改善。

**篩選用法：**
- `三率三升 年(1=是 0=否)` → `>=` → `1` ← 全年三率均升
- `三率三升 季(1=是 0=否)` → `>=` → `1` ← 單季三率均升

也可個別設條件（AND 邏輯）：

| 條件 | 意義 |
|------|------|
| `毛利率年增 YOY (pp) > 0` | 毛利率有改善 |
| `營益率年增 YOY (pp) > 0` | 本業獲利改善 |
| `淨利率年增 YOY (pp) > 0` | 整體獲利品質改善 |

### 操作步驟

1. **選擇股票池** - 選擇美股 / 台股上市 / 台股上櫃 / 自訂清單
2. **設定篩選條件** - 最多可設定 3 個條件（AND 邏輯）
   - 範例 1：`PE < 20`、`ROE > 15`、`殖利率 > 3`
   - 範例 2：`三率三升 年` >= `1`、`營收年增率 YOY > 0`
3. **點擊「篩選」** - 系統抓取基本面資料並套用條件
4. **查看結果** - 結果表格可排序，勾選感興趣的標的
5. **⬇ 下載 CSV** - 點擊「下載 CSV」儲存篩選結果（`screener_results.csv`）
6. **送入回測** - 勾選標的後點擊「送入回測 →」，自動跳轉至回測頁

### CSV 匯入格式

CSV 檔案需包含 `Symbol` 欄位，其餘欄位可為：
```
Symbol, Name, PE, PB, DividendYield, ROE, RevenueGrowth, EPS
```
也支援中文欄位名：`代碼, 名稱, 本益比, 股價淨值比, 殖利率, 股東權益報酬率, 營收成長率, 每股盈餘`

## 三、回測

### 資料來源設定

| 選項 | 說明 |
|------|------|
| 國際市場 (yfinance) | 輸入代碼即時下載，如 AAPL、ES=F |
| 已下載資料 | 從專案 `data/csv/` 資料夾讀取已下載的 CSV 檔案 |
| CSV 匯入 | 上傳自備的 OHLCV CSV 檔案 |

### 策略設定

#### 方向
- **做多 (Long)** - 買入後等待漲價賣出
- **做空 (Short)** - 賣出後等待跌價回補

#### 均線濾網 (進場前置條件)

在進場條件之前，可啟用均線濾網作為方向過濾器。濾網不通過時，即使進場條件全部滿足也不會進場。

| 濾網 | 說明 | 範例 |
|------|------|------|
| **均線位置濾網** | 收盤價在 MA/EMA 之上才能做多，之下才能做空 | 啟用 MA 20 → 價格在 20MA 上方才允許做多進場 |
| **均線方向濾網** | MA/EMA 方向持續上彎才能做多，下彎才能做空 | 啟用 MA 20 → 20MA 斜率為正才允許做多進場 |

- 兩個濾網可同時啟用（AND 邏輯）
- 均線類型可選 MA（簡單移動平均）或 EMA（指數移動平均）
- 週期可自由調整（預設 20）
- 啟用後，濾網 MA 會顯示在 K 線圖上（虛線）

#### 進場/出場條件設定

每條規則分為兩列顯示：

**第一列：** 指標 | 欄位（選完指標後自動顯示可用欄位）

**第二列：** 條件 | 值

**第三列：** 指標參數（選完指標後自動填入預設值，可手動修改）

| 欄位 | 說明 | 範例 |
|------|------|------|
| 指標 | 選擇技術指標 | RSI, MACD, MA, EMA, KD, Bollinger, ATR, Pivot, SRLevel, DoubleTop, DoubleBottom |
| 欄位 | 指標的輸出欄位 | RSI → `RSI`；MACD → `MACD_line` / `MACD_signal` / `MACD_hist` |
| 條件 | 比較運算子 | `>`, `<`, `>=`, `<=`, `crosses_above`, `crosses_below` |
| 值 | 門檻值或另一個欄位名 | 數字如 `30`，或欄位名如 `MACD_signal` |

- **進場條件**：AND 邏輯（全部滿足才進場）
- **出場條件**：OR 邏輯（任一滿足即出場）
- 點擊「+」按鈕可新增多條規則

### 技術指標參數預設值

| 指標 | 預設參數 | 輸出欄位 |
|------|---------|---------|
| MA (簡單移動平均) | period=20 | MA |
| EMA (指數移動平均) | period=20 | EMA |
| RSI (相對強弱指標) | period=14 | RSI |
| MACD (移動平均收斂擴散) | fast=12, slow=26, signal=9 | MACD_line, MACD_signal, MACD_hist |
| KD (隨機指標) | k_period=14, d_period=3 | K, D |
| Bollinger (布林通道) | period=20, std_dev=2.0 | BB_upper, BB_middle, BB_lower |
| ATR (平均真實範圍) | period=14 | ATR |
| Pivot (樞軸點) | pivot_type=1 | Pivot_P, Pivot_R1, Pivot_S1, Pivot_R2, Pivot_S2, Pivot_R3, Pivot_S3 |
| SRLevel (支撐壓力) | order=5, merge_pct=1 | SR_resistance, SR_support |
| DoubleTop (M頭) | order=10, tolerance_pct=3, min_bars=10 | DT_signal, DT_neckline, DT_target |
| DoubleBottom (W底) | order=10, tolerance_pct=3, min_bars=10 | DB_signal, DB_neckline, DB_target |

#### Pivot 樞軸點 — 參數說明

| pivot_type | 類型 | 特點 |
|-----------|------|------|
| 1 | Standard (Floor) | 經典公式，P=(H+L+C)/3 |
| 2 | Fibonacci | 基於 Pivot 加上 0.382/0.618 倍的前一根振幅 |
| 3 | Camarilla | 基於收盤價加減振幅乘數，適合日內交易 |
| 4 | Woodie | 加重收盤價，P=(H+L+2C)/4 |

輸出 7 條水平線：P（樞軸）、R1/R2/R3（壓力）、S1/S2/S3（支撐）。

#### SRLevel 支撐壓力 — 參數說明

| 參數 | 預設 | 說明 |
|------|------|------|
| order | 5 | 前後各幾根 K 棒判斷是否為局部極值（越大越嚴格） |
| merge_pct | 1 | 合併容差 (%)，相距 1% 內的極值合併為一個水平 |

- **SR_resistance**：當前收盤價上方最近的壓力位
- **SR_support**：當前收盤價下方最近的支撐位

#### DoubleTop M頭 / DoubleBottom W底 — 參數說明

| 參數 | 預設 | 說明 |
|------|------|------|
| order | 10 | 局部極值判斷嚴格度（前後各幾根 K 棒），越小越敏感 |
| tolerance_pct | 3 | 兩峰/兩谷價差容許百分比（以形態高度為基準） |
| min_bars | 10 | 兩峰/兩谷最少間隔 K 棒數 |

- `DT_signal` / `DB_signal`：1 = 形態確認，0 = 無信號
- `DT_neckline` / `DB_neckline`：頸線價位
- `DT_target` / `DB_target`：目標價

**圖表顯示：** M頭確認處顯示紅色倒三角 + "M" 標記；W底確認處顯示綠色三角 + "W" 標記。

### 風控設定

| 參數 | 預設值 | 說明 |
|------|--------|------|
| 停損 % | 無 | 虧損達此比例自動平倉 |
| 停利 % | 無 | 獲利達此比例自動平倉 |
| 部位比例 % | 100 | 每次交易使用多少比例的資金 |
| 初始資金 | 1,000,000 | 回測起始金額 |
| 手續費 % | 0.1 | 每筆交易的手續費率 |
| 滑價 % | 0.05 | 模擬實際成交與理論價格的滑價 |

### 回測結果

#### Chart（圖表）
- **K 線圖** - 顯示標的價格走勢，指標直接疊加
- **進出場標記** - 藍色三角（買入做多）、紅色三角（賣出放空）、橘色 X（平倉）
- **子圖** - RSI、MACD、KD、ATR 各自獨立顯示
- **成交量** - 底部顯示成交量柱狀圖
- **權益曲線** - 資金變化走勢 + 回撤百分比

#### Trades（交易明細）
- 每筆交易的進出場時間、價格、數量、損益、報酬率
- 獲利交易顯示綠色，虧損交易顯示紅色

#### Metrics（績效指標）

| 指標 | 說明 |
|------|------|
| 總報酬率 | 整段回測期間的總報酬 |
| 年化報酬率 | 換算為年度報酬 |
| Sharpe Ratio | 風險調整後報酬（越高越好） |
| 最大回撤 | 期間內最大虧損幅度 |
| 勝率 | 獲利交易占比 |
| 獲利因子 | 總獲利 / 總虧損 |
| 交易次數 | 總交易筆數 |
| 平均報酬 | 每筆交易的平均報酬率 |

## 常見策略範例

### 1. RSI 超賣反彈（做多）
- **進場**: RSI → RSI → 14 → `<` → `30`
- **出場**: RSI → RSI → 14 → `>` → `70`
- **停損**: 5%、**停利**: 10%

### 2. 均線黃金交叉（做多）
- **進場**: EMA(5) `crosses_above` EMA(20)

### 3. MACD 金叉（做多）
- **進場**: MACD → MACD_line → `crosses_above` → MACD_signal
- **出場**: MACD → MACD_line → `crosses_below` → MACD_signal

### 4. KD 超賣金叉（做多）
- **進場條件 1**: KD → K → `crosses_above` → D
- **進場條件 2**: KD → K → `<` → `20`
- **出場**: KD → K → `>` → `80`

### 5. 布林通道下軌反彈（做多）
- **進場**: Bollinger → BB_lower → `>` → Close（收盤跌破下軌）
- **出場**: Bollinger → BB_upper → `<` → Close（收盤突破上軌）

### 6. RSI 超賣 + 均線濾網（做多）
- **均線位置濾網**: 啟用，MA 60
- **均線方向濾網**: 啟用，MA 20
- **進場**: RSI → RSI → `<` → `30`
- **說明**: 只在多頭趨勢中做超賣反彈

### 7. Pivot S1 反彈（做多）
- **進場**: Pivot → Pivot_S1 → `>` → Close
- **出場**: Pivot → Pivot_R1 → `<` → Close

### 8. W底做多 / M頭放空
- **W底做多進場**: DoubleBottom → DB_signal → `>=` → `1`
- **M頭放空進場**: DoubleTop → DT_signal → `>=` → `1`
- **停損**: 5%

## 注意事項

1. **yfinance 資料限制**：分鐘級資料最多回溯 60 天，日線無限制
2. **台股全部篩選**：上市+上櫃共 1700+ 支，每支需抓基本面，需要數分鐘，建議先加篩選條件
3. **YOY 計算欄位**（GrossMarginYOY 等）需從財報計算，會額外增加 API 請求時間
4. **三率三升**：任一 margin YOY 資料缺失則標為 0（非 1）
5. **crosses_above / crosses_below**：比較前後兩根 K 棒，判斷是否發生交叉
6. **回測執行**：逐 K 棒模擬，以下一根 K 棒開盤價成交，已計入手續費和滑價
7. **同時只持有一個部位**：不支援加碼或同時持有多個方向的部位
